# WhatsApp Formatting Rules

## Character / Length Limits
- **Message**: No hard character limit; practical limit for forwarded messages is ~1,000 characters before readers drop off
- **Caption (media)**: 1,024 characters
- **Status**: 700 characters

## Formatting Syntax
- **Bold**: Wrap text in *asterisks* — `*bold text*`
- **Italic**: Wrap text in _underscores_ — `_italic text_`
- **Strikethrough**: Wrap text in ~tildes~ — `~strikethrough~`
- **Monospace**: Wrap text in ```triple backticks``` — ` ```code``` `
- **Line breaks**: Single line break creates a new paragraph; use blank lines to create visual breathing room
- **Lists**: No native list rendering — use dashes (–), dots (•), or numbered lines manually

## Hashtag Rules
- **Hashtags are not functional in WhatsApp** — they do not link to feeds or topics
- Omit hashtags entirely from WhatsApp content; they appear as plain text and reduce readability

## Content Rules
- Short, punchy messages forward more readily than long walls of text
- Self-contained content (no login required, no paywall) spreads faster
- Emotional content (humor, inspiration, useful tips) drives forwarding behavior
- Media attachments (image + caption) can accompany text for higher engagement
- Avoid external links as the primary content — many users do not click; embed the value in the message itself

## Group Context
- Content written for broadcast lists or groups should feel direct and personal, not like a social media post
- Avoid formal/corporate tone — conversational register performs best
- Voice notes are highly effective in WhatsApp but not applicable for text-only content
